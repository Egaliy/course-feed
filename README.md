# Course Feed

Минимальный MVP закрытой ленты курса:

- админ пишет посты в Telegram-бота;
- бот сохраняет текст, фото, видео, аудио и альбомы;
- сайт показывает ленту только по временной ссылке;
- команда `/link` создает ссылку на 1, 3 или 6 месяцев.

## Локальный запуск

```bash
cd course-feed
npm install
cp .env.example .env
npm start
```

В `.env` нужно указать:

```bash
BOT_TOKEN=...
ADMIN_IDS=328826190
PUBLIC_BASE_URL=http://localhost:3000
PORT=3000
COURSE_TITLE="Лента курса"
```

Для продакшена `PUBLIC_BASE_URL` должен быть реальным адресом сайта, например `https://example.ru`.

## Команды бота

`/start` проверяет доступ администратора.

`/link` показывает кнопки срока доступа и создает приватную ссылку.

Любое обычное сообщение от администратора публикуется в ленту. Поддерживаются текст, ссылки, фото, видео, аудио, голосовые, документы и альбомы.

## Ограничения MVP

Стандартный Telegram Bot API скачивает файлы до 20 MB. Для больших видео понадобится self-hosted Telegram Bot API server или отдельное хранилище/загрузка видео по другой схеме.

Хранилище MVP находится в `data/db.json`, медиа сохраняются в `public/uploads`. Для первых 20 студентов и одного поста в день этого достаточно. При росте проекта лучше перейти на PostgreSQL и S3-совместимое хранилище.

## VPS

Для запуска в России лучше взять VPS с Ubuntu у российского провайдера, поставить Node.js 22+, запустить приложение через `pm2` или systemd и поставить Nginx как reverse proxy с HTTPS.
